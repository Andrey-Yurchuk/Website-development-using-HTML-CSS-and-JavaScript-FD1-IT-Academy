Почему HTML считает “#BruceWillis” и другие такие значения цветом?

Как известно, в HTML при определении цвета используется шестнадцатеричная система счисления. В качестве алфавита шестнадцатеричной системы счисления используются цифры от 0 до 9 и шесть первых латинских букв – A, B, C, D, E, F. При переводе в десятичную систему буквы заменяются числами 10, 11, 12, 13, 14, 15 соответственно.

В случае же, если заданные в HTML символы не соотвествуют шестнадцатеричной системе (или они пропущены), то такие символы интерпретируются как 0. А дальше действует следующий алгоритм по определению цвета:

  1. заменяются все невалидные шестнадцатеричные символы на 0 (например, #CANDY интерпретируется HTML как 0ca0d0000000);
  2. полученные шестнадцатеричные символы (0ca0d0000000) разбиваются на три равные группы, при этом каждый компонент представляет соответствующую цветовую составляющую цвета RGB (в нашем примере с #CANDY это RGB (0ca0, d000, 0000));
  3. усекается каждая из разбитых групп на два символа с права. Конечный результат и определяет цвет (на примере с #CANDY это #ca0d00 или rgb(202, 13, 0)).

Таким образом, в HTML может быть много слов которые будут цвета “#BruceWillis” или “stevensegal”.
